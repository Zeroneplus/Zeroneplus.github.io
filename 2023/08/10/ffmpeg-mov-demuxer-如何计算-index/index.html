ffmpeg mov demuxer 如何计算 index
2023-08-10 21:50:22 # ffmpeg # mp4

最近在看 ffmpeg mov demuxer 的代码，发现 mov demuxer 在 read header 的时候就把所有 sample 的 pts、dts、pos 信息都计算出来，存在一个 entry 数组里。这篇文章就分析一下这个过程。

代码如下

具体计算过程：

  • 先把 ctts 展开，ctts 原来是游程编码形式，把所有 entry 都展开

  • 遍历每一个 chunk

    • 先计算当前 chunk 的 chunk offset

    • 计算当前 chunk 属于哪一个 sample to chunk

      • 从而获得当前 chunk 的 sample per chunk、当前 chunk 的 stsd index

    • 遍历当前 chunk 的每一个 sample

      • 判断当前 sample 是不是关键帧，通过当前 sample 的序号和 stss 中进行比对

      • 通过当前 sample 的序号找到 sample size

      • 为当前 sample 建立 index

        • 设置 e->pos，表示 sample 在文件中的位置

        • 设置 e->timestamp，表示 sample 的 dts

          • 该 dts 是通过累积 stts 来计算的

        • 设置 e->size，表示 sample 的 size

        • 设置 e->min_distance，表示当前 sample 离前一个参考帧的距离，seek 的时候会 seek 到参考帧中

上面就是建立 index 的过程，把 index 建立之后，后续 seek 就很方便了，下篇文章再分析 mov demuxer 中的 seek 过程。
