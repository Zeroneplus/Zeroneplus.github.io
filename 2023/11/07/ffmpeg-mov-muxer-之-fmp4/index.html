ffmpeg mov muxer 之 fmp4
2023-11-07 21:17:38 # ffmpeg # mp4

最近在使用 ffmpeg mp4 muxer 的时候遇到一个问题，当长时间录像时，由于 mp4 muxer 会在每写入一个 sample 之后，增加一个 entry 来存储这个 sample 的信息（用于 write_trailer 时得到 moov box），随着录像时间越来越长，相关 sample 的元数据信息就会越来越多，对于小内存的设备来说，就会出现 out of memory 的情况。

后来了解到 mp4 还存在一种 fragment mp4 模式，fmp4 就是把一个大的 mp4 文件分成几个小的部分，每个部分依次存储在一个文件中，每个部分的 moov/moof、mdat 邻近存储，可以在写完一定长度的数据之后就写入元信息，从而避免将元信息留到最后来写，这种模式相较 mp4 来说内存压力会小很多，特别是对于小设备且长期录像的场合。

从 mov muxer 的 option 中发现，存在一些和 fragment 相关的选项，比如 frag_duration，设定了这个选项之后，mov muxer 就会按照 frag_duration 的大小来进行分片，比如下面就设定分片时长为 5s（5000000 对应的 timescale 是 1000000）
