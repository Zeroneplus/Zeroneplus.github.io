这篇文章接着上一篇继续分析 ChunkDemuxer 相关的内容。

MediaSource::addSourceBuffer

先看 MediaSource::addSourceBuffer 的实现。

MediaSource::addSourceBuffer 调用 MediaSource::AddSourceBuffer_Locked

MediaSource::AddSourceBuffer_Locked 内部先通过 MediaSource::CreateWebSourceBuffer 创建 WebSourceBuffer 的对象，然后用该对象创建 SourceBuffer 的对象

MediaSource::CreateWebSourceBuffer

CreateWebSourceBuffer 函数通过 web_media_source_ 来进行 AddSourceBuffer，web_media_source_ 是 WebMediaSourceImpl 类型的对象，是在 WebMediaPlayerImpl::OnDemuxerOpened 中创建的

WebMediaSourceImpl::AddSourceBuffer

WebMediaSourceImpl::AddSourceBuffer 会在 ChunkDemuxer 内部创建一个 SourceBufferState，然后返回一个标识 id，用该 id 来创建 WebSourceBufferImpl

创建 StreamParser

ChunkDemuxer::AddId 先创建 StreamParser

CheckTypeAndCodecs 会检查输入的 type 和 codecs 信息，然后返回一个函数指针用来创建 StreamParser

kSupportedTypeInfo 数组的每一项对应了一个 codec 以及相应的函数

比如如果是 MP4 格式，那么就会使用 BuildMP4Parser 来创建 StreamParser

ChunkDemuxer::AddIdInternal

ChunkDemuxer::AddId 中创建完 StreamParser 后会调用 AddIdInternal 函数

AddIdInternal 内部先创建 FrameProcessor，然后用 FrameProcessor 和 StreamParser 去创建 SourceBufferState

SourceBufferState::Init

调用 SourceBufferState::Init 进行初始化。init_cb_ 将在 SourceBufferState::OnSourceInitDone 中触发，而 SourceBufferState::OnSourceInitDone 又被作为 StreamParser 的回调

SourceBufferState::InitializeParser 中会调用 MP4StreamParser::Init，并且将 SourceBufferState 的函数设置为 MP4StreamParser 的回调，目前重点关注 SourceBufferState::OnSourceInitDone、SourceBufferState::OnNewBuffers、SourceBufferState::OnEncryptedMediaInitData

init_cb_ 是在 MP4StreamParser::ParseMoov 中被调用的，也就是上层往 SourceBuffer 中 appendBuffer 后进行 Parse 之后可能会调用，后面会分析 appendBuffer 流程。

new_buffers_cb_ 是在 MP4StreamParser::Parse 中调用，该函数就是 SourceBufferState::OnNewBuffers，通过该函数往 FrameProcessor 中送数据，会面也会分析该过程。

encrypted_media_init_data_cb_ 也是在 MP4StreamParser::ParseMoov 中被调用，因为加密信息就在 moov 中的 stsd 中的 entry 里面。

这篇文章先分析到这里，下篇文章分析 SourceBuffer::appendBuffer 相关过程。
