接下来就具体看看每个函数做了什么工作。

av_guess_format

av_guess_format 函数返回一个 AVOutputFormat 指针，优先使用输入的 short_name 和 AVOutputFormat.name 进行比对，其次根据 mime_type，最后根据 filename 的后缀名决定使用哪个解封装格式。

avformat_alloc_output_context2

avformat_alloc_output_context2 函数创建 AVFormatContext 对象，其内部也有 av_guess_format 过程，在创建 AVFormatContext 时，会根据 AVOutputFormat 创建相应的私有结构，还会把属性设为默认值。

avformat_alloc_context

avio_open2 属于 IO 部分，这部分内容放在另一篇文章中介绍。

avcodec_find_encoder

avcodec_find_encoder 是根据输入的 AVCodecID 查找 AVCodec*，思路和上面的 av_guess_format 里面 AVOutputFormat 查找比较类似，这里不仔细展开

avcodec_alloc_context3

找到 AVCodec 后就可以创建 AVCodecContext，和上面通过 AVOutputFormat 创建 AVFormatContext 类似

avcodec_open2

创建 AVCodecContext 后还要进行打开操作，如果之前 avcodec_alloc_context3 创建 AVCodecContext 的时候已经传了 AVCodec，那么下面的 codec 参数可以为空，如果要传 codec，要注意和 AVCodecContext 中的一致（如果有的话）。avcodec_open2 会创建并初始化 AVCodecInternal

avformat_new_stream

为了将 encoder 和 muxer 联系起来，需要创建 stream，在创建 stream 的时候，也会在内部创建 AVCodecContext，但是 stream 内部的 AVCodecContext 已经废弃了，所以上面的例子中单独创建了 AVCodecContext

avcodec_parameters_from_context

把之前单独创建的 AVCodecContext 中的参数拷贝到 stream 中的 codecpar

avformat_write_header

对 AVFormatContext 做一些初始化工作。在 avformat_init_output 中可能会执行 init_pts 函数，所以在 avformat_write_header 中做了一个特判，避免重复调用 init_pts

avio_write_marker 就是将当前 iostream 标记为某个类型，可以看到，在调用 ffofmt(s->oformat)->write_header 后又调用 avio_write_marker 将当前 iostream 标记为 AVIO_DATA_MARKER_UNKNOWN
