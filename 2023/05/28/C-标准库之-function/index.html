这篇文章接着上一篇分析 function。

示例代码

function 可以把函数、可调用对象等包装为新的可调用对象。还是先看例子（来自 cppreference）

可以看到，function 的实例可以存储、拷贝、调用任何可以拷贝构造的可调用对象：

  • 函数（会 decay 为函数指针）
  • lambda 表达式返回值
  • bind 函数返回值
  • 重载了调用运算符的类的实例
  • 类成员函数指针
  • 类成员数据指针

注意如果给 function 的参数是类成员函数指针，那么调用时第一个参数必须是对象，要满足类成员函数指针的调用要求。

function

先看 function 类

function 只有 function<_Rp(_ArgTypes...)> 这个特化版本是有定义的，其他都是未定义的。

function 可以使用不同的 __func，这里先只分析 __value_func 作为 backend 的情况

接受模板参数的构造函数

function 有一个接受模板参数的构造函数，正是这个构造函数，function 才能接受函数指针、lambda 表达式返回值、bind 返回值作为参数

注意这个构造函数参数没有带引用，所以模板类型会推断为值类型，比如输入参数是一个函数名，_Fp 会推断为函数指针类型，比如下面的例子：

输入参数是函数名，所以 _Fp 推断为函数指针。由于是传值，这里要发生值的构造，但如果传的参数是右值引用，实参是值类型，那么 __f 会发生移动构造。在构造 __f_ 时，把实参 __f 进行了 move，所以构造 __f_ 时传的是右值引用。

_EnableIfCallable

看看 _EnableIfCallable 这个条件

他要求 _Fp 满足：首先 _Fp 不是当前 function 类型，并且对于给定 _ArgTypes，_Fp& 可以调用，其次要求

_Rp 是 void，或者 _Fp 的返回值可转换到 _Rp 类型。

__invokable 在 bind 里面已经介绍过了，这次看看 __invoke_of：在 __invokable 满足的条件下，通过 __invokable_r 得到返回值，__invokable_r 在 bind 里面也介绍了

__value_func 中接受右值引用的模板构造函数

再看 function 内部的 __f_，__f_ 是 __function::__value_func 类型

function 内部构造 __value_func 传的是右值引用，先看接受右值引用的模板构造函数

__value_func(_Fp&& __f) 内部调用了 __value_func(_Fp&& __f, const _Alloc& __a)。

在上面函数中，真正存储可调用对象的是 __function::__func<_Fp, _Alloc, _Rp(_ArgTypes...)> 的实例。如果 __function::__func 的 size 较小，就使用 __value_func 内部的 __buf_，否则就新申请内存。

__function::__func

__function::__func<_Fp, _Alloc, _Rp(_ArgTypes...)> 继承 __function::__base<_Rp(_ArgTypes...)>，__base 是虚基类

__function::__func 内部还有 __alloc_func，__alloc_func 才是真正存储可调用对象的地方

__alloc_func

在 __alloc_func 内部有一个 __compressed_pair 来存储可调用对象和 allocator，__compressed_pair 之前已经介绍过了

__alloc_func 内部使用了 __compressed_pair 的 piecewise_construct 函数，使用了之前说过的 forward_as_tuple，通过他得到的 tuple 里面的类型是引用类型

通过上面的分析可知，在创建 function 对象时，最开始 function 的构造函数内由于传参是值类型，所以发生了值的构造，但是后面都是尽可能以右值引用的方式往下传参数，最后在 __alloc_func 内部，也就是真正保存可调用对象的地方是按值存储可调用对象的，但是构造该对象时传的是右值引用，所以会尽可能使用移动构造，把最开始 function 构造函数中创建的值的资源转移到 __alloc_func 的 __compressed_pair 的值中。

当然上面说的是最理想的情况，如果可调用对象没有定义移动语义还是会拷贝构造。

function::operator()()

注意 operator()()的参数不是模板类型，就是定义 function 时设定的模板类型，这和 bind 不一样。但是 _ArgTypes 中可以出现引用，比如 function<void(int&)>，因为函数形参可以是引用类型

__value_func::operator()()

__f_ 是 __value_func 类型，所以 function::operator()()内部调用 __value_func::operator()()

__function::__func::operator()()

__f_ 是 __base<_Rp(_ArgTypes...)> 类型的指针，但是实际指向 __function::__func，__function::__func 继承 __function::__base，而 __function::__func::operator()() 还是参数转发，交给 __alloc_func::operator()()

__alloc_func::operator()()

__alloc_func 内部取出可调用对象，进行调用

__invoke_void_return_wrapper

__invoke_void_return_wrapper 如下，内部调用了 __invoke，__invoke 之前已经介绍

拷贝构造

内部发生 __value_func 的拷贝构造

__value_func

__value_func 内部调用 __func 的 clone 函数，由于 __value_func.__f_ 指针指向的内存可能不是动态分配的，所以这里存在两种 clone 方式，一种输入指定地址，在指定地址上构造 __func，另一种则是直接动态分配 __func 的内存，然后构造。下面很多函数都针对这个机制做了不同处理

__func::__clone

__alloc_func

真正的构造发生在 __alloc_func 中，因为 __alloc_func 有 __compressed_pair 真正存储可调用对象

移动构造

调用 __value_func 的移动构造

__value_func
