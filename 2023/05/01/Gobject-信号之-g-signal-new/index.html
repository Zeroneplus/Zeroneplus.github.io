Gobject 信号之 g_signal_new
2023-05-01 20:35:40 # gobject

这篇文章来介绍 Gobject 的最后一个特性：信号和闭包。

Gobject 的信号主要有 3 个 api：g_signal_new、g_signal_connect、g_signal_emit。使用流程如下：

  • 首先在 class_init 中调用 g_signal_new 创建信号，这里要注意，信号是属于类的，和属性一样
  • 通过 g_object_new 创建对象，然后将回调函数通过 g_signal_connect 连接到信号上
  • 最后对对象调用 g_signal_emit 触发信号，执行上一步连接的回调函数

g_signal_new

  • signal_name。信号的名字
  • itype。信号所属的类型，注意信号可以继承，子类可以使用父类的信号
  • signal_flags。指定默认 handler（通过 class_offset 指定的虚函数）的属性，可以是 G_SIGNAL_RUN_FIRST、G_SIGNAL_RUN_LAST 等
  • class_offset。类结构 struct 实例的首地址加上该偏移指向一个函数指针，这个函数指针指向的函数就是信号的默认 handler，子类可以通过虚函数覆盖（修改指定偏移处的函数指针）来定义自己的默认 handler，如果 class_offset 为 0，那么将没有默认 handler
  • accumulator。在每执行一个 handler 后会执行 accumulator，会根据其返回值决定是否继续执行剩下的 handler
  • accu_data。传给 accumulator 函数的数据
  • c_marshaller。自定义的 marshal 函数，内部需要执行 callback 函数，建议保持为空，使用默认 marshal 函数即可
  • return_type。返回值类型
  • n_params。去除第一个 GObject 参数后剩余的参数个数，不考虑 user pointer 在内
  • 变长参数。去除第一个 GObject 参数后剩余的参数的类型，不考虑 user pointer 在内

g_signal_type_cclosure_new

当 class_offset 不为 0 时，会调用 g_signal_type_cclosure_new 函数创建一个 GClosure，这个 GClosure 用于处理默认 handler。

  • g_signal_type_cclosure_new 先创建了一个 GRealClosure，这个 GRealClosure 里面有 GClosure
  • 然后将 itype 也就是 GType 类型设给了 GClosure.data，这样就知道这个 Closure 只能在 itype 及其子类的对象中执行，后面会进行这个检查
  • 然后将 struct_offset 也就是虚函数的偏移设给 GRealClosure.meta_marshal_data
  • 然后将 g_type_class_meta_marshal 函数设给了 GRealClosure.meta_marshal
  • 然后将 g_type_class_meta_marshalv 函数设给了 GRealClosure.va_meta_marshal

上面的 itype、struct_offset、g_type_class_meta_marshal、g_type_class_meta_marshalv 在执行默认 handler 时会用到，比如会通过 struct_offset 信息找到需要执行的函数在虚函数表中的位置

g_closure_new_simple

创建 GRealClosure，然后设置 GRealClosure 中 GClosure 部分的 data 字段的值

GRealClosure 和 GClosure 的关系如下，GRealClosure 在 GClosure 的基础上多了一些私有变量

g_closure_set_meta_marshal

这个函数把输入的 meta_marshal、marshal_data 设给了 GRealClosure

g_closure_set_meta_va_marshal

这个函数把输入的 va_meta_marshal 设给了 GRealClosure

g_signal_new_valist

g_signal_new 在创建默认 handler 的 Closure 后执行 g_signal_new_valist。

g_signal_new_valist 主要是将变长参数整理为 GType 数组，然后调用 g_signal_newv

g_signal_newv

像之前的属性一样，g_signal_newv 也会创建一个信号对象，然后把这个对象插入一个全局的 hash 表中。下面的函数虽然长，但其实做的事主要是：

  • 创建 SignalNode 对象，并把他插入一个全局的表中
  • 把输入的 class_closure、accumulator、return_type、param_types 等信息复制到 SignalNode 中

后面只要有这个 SignalNode，那么完全就知道 handler 函数该怎么调用了
