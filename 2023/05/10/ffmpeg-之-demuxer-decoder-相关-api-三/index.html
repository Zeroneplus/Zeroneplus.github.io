avformat_find_stream_info 函数最复杂，后面基本上就好理解了。

av_find_best_stream

  • type 表示想要的是 audio 还是 video stream
  • wanted_stream_nb 想要的 stream 的序号
  • related_stream 设置此项表示返回的 stream 要和 related_stream 相关联，比如都处于同一个 AVProgram
  • decoder_ret 返回该 stream 的 AVCodec

如果设置了 related_stream，优先在这个 stream 对应的 program 中查找，如果没找到，则在全体 stream 中查找

avcodec_receive_frame

优先返回 avci->buffer_frame，如果 avci->buffer_frame 为空，调用 decode_receive_frame_internal 进行解码

avcodec_send_packet

和 avcodec_send_frame 一样，avcodec_send_packet 也是先将输入的 avpkt 引用到 avci->buffer_pkt，然后再把 buffer_pkt 送到 avci->bsf，最后如果 buffer_frame 为空，尝试解码。

av_bsf_send_packet

这一步是将 buffer_pkt 送到 bitstream filter 中，经过 bitstream filter 后再解码。

这里没有调用 bitstream filter 对 packet 进行处理，真正处理是在下面 decode_receive_frame_internal 时从 bitstream filter 取数据时会进行 bitstream 滤波处理。

这一步其实是把 avci->buffer_pkt 转移到 bsfi->buffer_pkt

再看看这个函数。可以看到，优先从 packet_buffer 中取数据，这个就是之前 find_stream_info 时如果没有 AVFMT_FLAG_NOBUFFER 这个 flag，会把读取的数据存到 packet_buffer。当 packet_buffer 没有缓存时，也是调用 read_frame_internal 读取数据
