Gobject初始化之类结构struct初始化
2023-04-29 19:48:00 # gobject

上篇博客介绍了类型注册的过程，类型注册的结果就是创建并初始化类对应的 TypeNode 实例。之前在介绍如何定义一个 Gobject 类时提到了类的 结构struct，这个 struct 每个类全局只有一个实例，用于实现类似 C++ 虚函数覆盖的效果。这篇博客就介绍一下类结构 struct 的初始化过程，同时通过代码分析一下 base_init、class_init 等函数的调用时机。

当通过 g_object_new 创建 实例struct 的实例时，如果类 结构struct 的实例没有被创建，会先创建该实例

g_object_new

简单起见，假设上面函数调用时没有传递属性参数，于是调用的是 g_object_new_with_properties()

g_type_class_peek_static

g_object_new_with_properties 先调用 g_type_class_peek_static 尝试得到 结构struct 的实例

g_type_class_ref

如果 g_type_class_peek_static 返回的是空指针，代表此时还没有创建结构 struct 的实例，g_object_new_with_properties 会调用 g_type_class_ref 函数创建类结构 struct 的实例。g_type_class_ref 函数会：

Increments the reference count of the class structure belonging to @type. This function will demand-create the class if it doesn’t exist already.

由于 g_type_class_ref 和 g_object_new_internal 都会增加引用计数，所以引用计数会 +2，于是 g_object_new_with_properties 的最后还做了一个 unref 动作

type_data_ref_U

g_type_class_ref 函数先通过 type_data_ref_U 函数增加当前 TypeNode 的引用计数

如果 type_data_ref_U 函数失败，表明 TypeNode 可能没有初始化，此时将调用 type_data_ref_Wm 函数初始化 TypeNode，和之前的类型注册过程一样

type_class_init_Wm

TypeNode 初始化好之后，调用 type_class_init_Wm 函数创建结构 struct 的实例，该实例的指针会保存在 TypeNode->data->class.class 中

上面这个函数主要：

  • 为当前要创建的结构 struct 的实例分配内存
  • 如果有父类，就把父类的结构 struct 的实例复制到当前的结构 struct 的实例
  • 从最顶层父类到自己，依次调用每一个 base_init 函数初始化当前的结构 struct 的实例
  • 调用当前类的 class_init 函数初始化当前的结构 struct 的实例

此外还可以发现创建的结构 struct 的实例中还含有与之关联的 GType 信息，那些类型检查宏会用到这些信息，这部分后面再单独写一篇文章介绍

关于上面的‘private size’：

  • class_private_size 是通过 g_type_add_class_private 函数修改
  • 而 instance.private_size 是通过 g_type_class_adjust_private_offset 函数修改

class_private_size 用得比较少，instance.private_size 就是之前博客介绍的类的私有结构的 size，在 type_class_init_Wm 函数中会把父类的 instance.private_size 赋给子类，子类如果也想定义自己的私有结构，会在 class_intern_init 函数中调用 g_type_class_adjust_private_offset 中增加 instance.private_size。

g_object_base_class_init

最后看一下 GObject 的 base_class_init 函数

该函数主要是做了一些拷贝工作，避免父子类中的指针指向同一个对象（因为创建结构 struct 的实例时会直接拷贝父类的结构 struct 的实例），比如把 construct_properties 复制一份，把 pspecs 置为空。base_init 大部分就是做这种工作，比如 gstreamer 中的 GstElement 的 base_init 就是拷贝 padtemplates，让父子类的 padtemplate 指向不同的对象，避免析构时多次释放。
