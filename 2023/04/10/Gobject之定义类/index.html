Gobject 是一个 C 语言库，它希望用 C 实现类似 C++ 的面向对象、继承等特性。

几大特点：

  • 类似 C++ 的面向对象、继承、虚函数机制
  • 基于引用计数的内存管理
  • get、set 属性设置
  • 闭包和回调函数机制

第三步，定义两个 struct：一个 实例struct，一个 类struct。实例struct 存储实例数据；类struct 保存函数指针，用于实现类似 C++ 的虚函数覆盖机制。每一个类都有唯一的 类struct 的实例，而每一个类的实例则分别对应一个 实例struct 的实例。

上面第三步在定义 MyObj 和 MyObjClass 时，他们的第一个成员分别是 GObject 和 GObjectClass。Gobject 库通过这种方式实现继承：当子类要继承一个父类时，会把父类的 实例struct 和 类struct 的 实例 作为自己的 实例struct 和 类struct 的第一个成员，这样子类内部相当于内嵌了一个父类（和 C++ 的继承很像）。

Gobject 的命名约定：

  • Gobject 约定项目名和类名都是首字母大写，其余字母小写，比如上面的 My 和 Obj。
  • Gobject 约定 实例struct 名字是 项目名类名，类struct 名字是 项目名类名Class，遵循驼峰命名规则，比如上面 MyObj、MyObjClass。
  • Gobject 约定函数前缀是全小写的 项目名_类名，用下划线分隔，比如 my_obj。上面例子中的 init 和 class_init 函数全名分别是 my_obj_init、my_obj_class_init。

把宏展开之后就好理解了，G_DEFINE_TYPE 宏帮我们定义了一些函数，比如定义 my_obj_get_type 函数用于类型注册，同时还声明了 init、class_init 函数。

这里简单的介绍一下 Gobject 类的初始化过程：

  • 当用户使用类型标识宏（比如 MY_TYPE_OBJ）时，由于类型标识宏实际就是一个函数调用，所以会调用具体函数，上面例子中就是调用 my_obj_get_type 函数，my_obj_get_type 函数会调用 my_obj_get_type_once 函数执行类型注册，这是一个 thread_once 的过程。
  • 注册类型过程中会创建 类struct（上面例子中是 MyObjClass）的实例，同时执行类初始化函数 my_obj_class_intern_init，在 my_obj_class_intern_init 中调用用户定义的类初始化函数 my_obj_class_init。
  • 创建 类struct 的实例后，才能创建 实例struct（上面例子中是 MyObj）的实例，创建实例后会调用用户定义的实例初始化函数 my_obj_init 进行初始化。
